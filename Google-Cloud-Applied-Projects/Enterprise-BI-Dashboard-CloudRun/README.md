# 📈 Enterprise BI Dashboard (Cloud Run Serverless Deployment)

A high-performance, fully cloud-hosted **Business Intelligence (BI)** dashboard designed to simulate and analyze enterprise-scale ERP and Supply Chain transactional data. Built using **Streamlit**, **Pandas**, and **Plotly**, and deployed natively to the internet via **Google Cloud Run** using a custom Docker containerized architecture.

---

## 🎯 Strategic Value Proposition & Enterprise Impact

In the modern data ecosystem, building local analytical models is insufficient; insights must be democratized and accessible to stakeholders anywhere, instantly. This project bridges the gap between raw data analysis and scalable enterprise deployment. 

By utilizing a **Serverless Cloud Architecture**, this dashboard ensures:
1.  **Zero-Downtime Accessibility**: Executives can access live KPIs (Total Sales, Profit Margins, Regional Dominance) via a secure, public URL without needing localized software.
2.  **Infrastructure as Code (IaC)**: The entire deployment environment is defined within a `Dockerfile`, ensuring the application can be seamlessly replicated, scaled, or audited by DevOps teams.
3.  **Real-Time Data Synthesis**: Automatically ingests simulated raw ERP logs and transforms them into interactive, executive-ready visual insights.

---

## 🛠️ Technical Architecture & Cloud Infrastructure

-   **Containerization (`Dockerfile`)**: The application is packaged within a lightweight, optimized `python:3.11-slim` image, ensuring minimal memory footprint and rapid spin-up times on the cloud server.
-   **Serverless Deployment (Google Cloud Run)**: The container is pushed to Google Artifact Registry and deployed via Cloud Run, which automatically scales compute resources up during heavy traffic and scales down to zero when idle, optimizing operational costs.
-   **Identity & Access Management (IAM)**: Configured custom service account bindings (`roles/storage.admin`) via the Google Cloud CLI to authorize secure interactions between the build pipeline and the cloud storage buckets.

🚀 Cloud Deployment Pipeline (Reproducibility)

To replicate this deployment on any authenticated Google Cloud Project:

    Initialize Environment:
    Bash

    git clone [https://github.com/YOUR_USERNAME/cloud-bi-deployment.git](https://github.com/YOUR_USERNAME/cloud-bi-deployment.git)
    cd cloud-bi-deployment

    Authorize Build Services:
    Ensure the GCP project has the required APIs enabled for container building:
    Bash

    gcloud services enable artifactregistry.googleapis.com cloudbuild.googleapis.com run.googleapis.com

    Resolve Service Account Permissions (IAM):
    Grant the default compute engine service account the necessary storage administrative rights to manage the built artifacts:
    Bash

    gcloud projects add-iam-policy-binding YOUR_PROJECT_ID \
        --member=serviceAccount:YOUR_PROJECT_NUMBER-compute@developer.gserviceaccount.com \
        --role=roles/storage.admin

    Execute Serverless Deployment:
    Command Google Cloud to build the Dockerfile and deploy the container to a public endpoint:
    Bash

    gcloud run deploy enterprise-bi-dashboard --source . --region europe-west1 --allow-unauthenticated
